🔧 エンジン＆トランスミッション CORE

RPM（回転数）ベースのエンジンモデル。トルクカーブ、ギア比、ファイナルドライブまで実車に準拠した計算。

こだわりポイント: AnimationCurveで定義したトルクカーブは5つのキーポイントで構成。低回転域ではトルクが立ち上がり、中回転でピーク、高回転で自然に落ち込む実車特性を再現。プレイヤーはパワーバンドを意識したシフト操作が求められます。

トルクカーブ (AnimationCurve)

5点のキーフレームで定義。RPM正規化値(0-1)に対するトルク倍率:

  • 0% (0 RPM) → 倍率 30%
  • 35% (2,450 RPM) → 倍率 80%
  • 55% (3,850 RPM) → 倍率 100% (ピーク)
  • 80% (5,600 RPM) → 倍率 95%
  • 100% (7,000 RPM) → 倍率 70%
driveTorque = maxTorque × curve(rpm/maxRPM)
    × gearRatio × finalDrive × throttle
wheelTorque = driveTorque / wheelRadius

5速ギアボックス

  • ギア比: 3.5 / 2.3 / 1.7 / 1.3 / 1.0
  • ファイナルドライブ: 3.8
  • 速度域: 65 / 100 / 135 / 176 / 230 km/h
  • シフトクールダウン: 0.3秒
rpm = wheelRPM × gearRatio × finalDrive
rpm = Clamp(rpm, idleRPM, maxRPM)
// wheelRPM = speed / (2π × radius) × 60

🛒 タイヤ摩擦モデル PHYSICS

WheelColliderのFrictionCurveを詳細設定。前輪と後輪で異なる特性を持たせ、実車のようなアンダー/オーバーステア特性を再現。

こだわりポイント: 前輪は横グリップを高めに設定しレスポンスの良い初期旋回を実現。後輪はstiffnessを1.25倍に設定することで、限界域では後輪が先に滑り出すリアルなオーバーステア特性を再現。この前後差がドライビングの「手応え」を生みます。

前輪フリクション

  • 前方: extremumSlip=0.14, value=1.15
  • 前方: asymptoteSlip=0.35, value=0.95
  • 横方: extremumSlip=0.22, value=1.45
  • 横方: asymptoteSlip=0.55, value=1.30
  • 横方stiffness: 1.0

後輪フリクション

  • 前方: 前輪と同一カーブ
  • 横方: 前輪と同一カーブ
  • 横方stiffness: 1.25 (後輪を硬めに設定)

後輪の横stiffnessを25%増すことで、高荷重時に後輪が安定し、低荷重時（ブレーキング等）に自然なオーバーステアを発生。

🛡 安全制御システム ASSIST

3つの独立した車両安定化システム。各システムは個別にON/OFF可能。

こだわりポイント: ESCは実車同様「理想ヨーレート」と「実際のヨーレート」の差分を監視。ホイールベースと速度から物理的に正しい理想値を計算し、オーバーステア/アンダーステアそれぞれに異なるブレーキ制御を適用。OFFにすればドリフト走行も可能です。

ABS（アンチロックブレーキ）

スリップ閾値 0.06 を超えるとブレーキ圧を15Hzでパルス制御。

slip = (wheelSpeed - groundSpeed) / max(groundSpeed)
if |slip| > 0.06:
  brake = 0 // パルスOFF
// 15Hz周期でON/OFF切替

TCS（トラクションコントロール）

駆動輪のスリップが 0.12 を超えるとモータートルクを低減。

if driveSlip > 0.12:
  motorTorque *= 0.3
// ステアリング角で閾値を動的拡大
threshold += |steerAngle| × 0.02

ESC（横滑り防止）

idealYaw = steerAngle × speed / wheelBase
error = actualYaw - idealYaw
deadzone = 0.2 rad/s

// オーバーステア: 外側前輪にブレーキ
// アンダーステア: 内側後輪にブレーキ
correction = maxBrake × 0.30 × |error|

アンチロールバー

antiRoll = (travelL - travelR) × force
Front: force = 5,500 N·m/rad
Rear: force = 3,500 N·m/rad

左右のサスペンション伸縮差からロール対抗力を算出。コーナリング時の車体傾斜を抑制。

Source: locaCarcontroller.cs — ESC (横滑り防止) 
private void ApplyESC(float steerInput, float speedMS)
{
    float yawRate = rb.angularVelocity.y;

    // 自転車モデル: 理想ヨーレート = (速度 × tan(舵角)) / ホイールベース
    float wheelbase = 2.5f;
    float steerRad = currentSteerAngle * Mathf.Deg2Rad;
    float expectedYawRate = (speedMS * Mathf.Tan(steerRad)) / wheelbase;

    float yawError = yawRate - expectedYawRate;

    // ステアリング量に応じてデッドゾーンを拡大 (通常旋回を誤検知しない)
    float steerRat = Mathf.Abs(currentSteerAngle)
        / Mathf.Max(lowSpeedSteerAngle, 1f);
    float escDeadZone = Mathf.Lerp(0.3f, 0.8f, steerRat);

    if (Mathf.Abs(yawError) > escDeadZone)
    {
        float correctionBrake = (Mathf.Abs(yawError) - escDeadZone)
            * escStrength * maxBrakeTorque * 0.25f;

        if (yawError > 0f)
            // オーバーステア: 外側前輪にブレーキ
            target = (steerInput >= 0f) ? leftFrontWheel : rightFrontWheel;
        else
            // アンダーステア: 内側後輪にブレーキ
            target = (steerInput >= 0f) ? rightRearWheel : leftRearWheel;

        if (target != null) target.brakeTorque += correctionBrake;
    }
}

⚙ 高度タイヤ物理 & 触覚フィードバック ADVANCED

Dev Note: タイヤリラクゼーションとは、タイヤが滑り始めてから最大グリップに達するまでの遅延を模擬するモデル。実車のタイヤはゴムの弾性変形に時間がかかるため、ステアリングを切った瞬間にはグリップが発生せず、数十ミリ秒後にピークに達する。これがないと操作がデジタル的で、「タイヤが路面を掴む感覚」が失われる。
参考文献: "Race Car Vehicle Dynamics" - Milliken & Milliken, SAE International, Chapter 2: Tire Behavior

👁 ルックアヘッド（先読み）システム LOOK-AHEAD

速度・コース逸脱度・カーブ曲率の3要素でルックアヘッド距離を動的に決定。

lookAheadSec = Lerp(0.3, 0.4, speedKmh / 180)
↓ コース逸脱時
lookAheadSec ×= Clamp01(1 - (deviation - 1.5) / 8) + 0.25
↓ オフトラック時
lookAheadSec ×= 0.4
↓ ポイント数に変換

カーブ曲率検出アルゴリズム

ルックアヘッド範囲内を2ポイントおきにサンプリングし、隣接セグメント間の角度変化を計測。

maxAngleChange > 20° のとき:
curveFactor = Lerp(1.0, 0.5, Clamp01((maxAngle - 20) / 40))
lookAheadPts = Max(3, Round(lookAheadPts × curveFactor))

20°で100% → 40°で75% → 60°以上で50%まで短縮。S字コーナーやシケインの事前検知に使用。

  • deviation > 4m: 速度ペナルティ発動

void BuildRacingLineSpline()
{
    // 1) Catmull-Romスプラインでセンターライン生成
    for (int i = 0; i < count; i++) {
        Vector3 p0 = waypoints[((i-1)+count) % count].position;
        Vector3 p1 = waypoints[i].position;
        Vector3 p2 = waypoints[(i+1) % count].position;
        Vector3 p3 = waypoints[(i+2) % count].position;
        for (int j = 0; j < splineResolution; j++) {
            float t = (float)j / splineResolution;
            centerLine.Add(CatmullRom(p0, p1, p2, p3, t));
        }
    }

    // 2) 各点の曲率→イン側にオフセットしてレーシングラインへ
    float curvature = Vector3.Cross(dirPrev, dirNext).y;  // 正=左、負=右カーブ
    Vector3 normal = new Vector3(tangent.z, 0f, -tangent.x);
    float curvatureStrength = Clamp(Abs(curvature) * splineResolution * 8f, 0, 1);
    float offset = Sign(curvature) * curvatureStrength * roadWidth * 0.4f * racingLineOffset;
    Vector3 racingPoint = centerLine[i] + normal * offset;

    // 3) Laplacianスムージング(3回): ジグザグ防止
    smoothed[i] = splinePoints[i]*0.5f + splinePoints[prev]*0.25f + splinePoints[next]*0.25f;
}

Catmull-Romスプライン公式

P(t) = 0.5 × [
  (2×P1) +
  (-P0 + P2) × t +
  (2P0 - 5P1 + 4P2 - P3) × t² +
  (-P0 + 3P1 - 3P2 + P3) × t³
]

4つの制御点(P0-P3)とパラメータt(0-1)から滑らかな曲線上の点を算出。splineResolution=10で各ウェイポイント間を10分割。

strength = Clamp(|curv| × 10 × 8, 0, 1)
offset = sign(curv) × strength
      × roadWidth × 0.4 × 0.6

void CalculateSteering()
{
    Vector3 targetPoint = CalculateRacingLineTarget(speedKmh);
    float steerAngle = Vector3.SignedAngle(forward, (targetPoint - pos).normalized, Vector3.up);

    // 非線形パワーカーブ: 小角度=穏やか、大角度=シャープ
    float normalizedAngle = steerAngle / 90f;
    float curvedValue = Mathf.Pow(Mathf.Abs(normalizedAngle), steeringCurve);  // 0.7
    float targetSteering = Mathf.Clamp(sign * curvedValue * steeringSensitivity, -1f, 1f);

    // 速度に応じた補間速度（高速=滑らか、低速=即応）
    float speedFactor = Mathf.Lerp(1.5f, 0.6f, Mathf.InverseLerp(30f, 150f, speedKmh));
    float smoothFactor = Mathf.Lerp(1f, 0.4f, Mathf.Abs(targetSteering - currentSteering));
    currentSteering = Mathf.Lerp(currentSteering, targetSteering,
        Time.deltaTime * steeringSmoothing * speedFactor * smoothFactor);

    // オーバーテイク: 直線〜緩カーブ(20°未満)でバイアスを加算
    if (overtakeSteeringBias != 0f && absAngle < 20f)
        currentSteering += overtakeSteeringBias * (1f - absAngle / 20f);
}

void DetectSlipAndLearn()
{
    // 全輪のスリップ量(前後+横)の平均を計算
    float totalSlip = 0f; int wheelCount = 0;
    foreach (AxleInfo axle in axleInfos) {
        if (axle.leftWheel.GetGroundHit(out hit)) {
            totalSlip += Mathf.Abs(hit.sidewaysSlip) + Mathf.Abs(hit.forwardSlip);
            wheelCount++;
        }
        // rightWheelも同様...
    }
    currentSlipAmount = totalSlip / wheelCount;

    // スリップが閾値超え → このウェイポイントの速度係数を下げる（学習）
    if (currentSlipAmount > slipThreshold)
        learnedSpeedModifiers[idx] = Mathf.Max(0.6f, learnedSpeedModifiers[idx] - learningRate);
    // 安定走行中 → 少しずつ速度を戻す（10倍遅い速度で回復）
    else if (currentSlipAmount < slipThreshold * 0.3f)
        learnedSpeedModifiers[idx] = Mathf.Min(1f, learnedSpeedModifiers[idx] + learningRate * 0.1f);
}

float EstimateCurveRadius(Vector3 a, Vector3 b, Vector3 c)
{
    Vector2 p1 = new Vector2(a.x, a.z);
    Vector2 p2 = new Vector2(b.x, b.z);
    Vector2 p3 = new Vector2(c.x, c.z);

    float la = Vector2.Distance(p2, p3);
    float lb = Vector2.Distance(p1, p3);
    float lc = Vector2.Distance(p1, p2);

    float s = (la + lb + lc) / 2f;                  // 半周長
    float area = Mathf.Sqrt(s * (s-la) * (s-lb) * (s-lc)); // ヘロン
    float radius = (la * lb * lc) / (4f * area);    // 外接円半径

    return Mathf.Clamp(radius, 5f, 1000f);
}

設計意図

なぜPlayOneShotか: Playだと前の音が止まるが、PlayOneShotは重複再生可能。多重クラッシュ時に音が途切れない。

なぜInverseLerpか: 線形マッピングだと低速衝突の音量変化が不自然。InverseLerpで閾値内を0-1に正規化し、Lerpで最小音量0.2を保証。完全な無音衝突を避ける。

envelope = (1 − t)²
thunk = sin(t × π × 1.5)
output = envelope × (thunk × 0.9 + noise)

各バンプは8ms幅。二次関数エンベロープで急速に減衰する「ドン」感。低周波サイン（1.5半周期）で重い打撃音を生成。ランダムノイズ(±0.15)で質感を付与。

速度連動ピッチ

pitch = Lerp(0.25, 0.8, speed / 35)

低速: 0.25 → ゆっくりした「ドン…ドン…」
高速: 0.8 → 速い「ドドドドド」

30バンプ/秒のクリップをピッチ変調で速度感を表現。ピッチ0.25で実効7.5回/秒（低速の縁石感）、0.8で24回/秒（高速のガタガタ感）。ピッチを低めに抑えることで重厚な「サンク」音質を維持。

🕹 デッドゾーン＆入力リマップ INPUT

ゲームパッドの入力をそのまま使うと微小な傾きでも車が動いてしまう。デッドゾーンと再マッピングで「ニュートラルの静寂」と「動き出しの滑らかさ」を両立。

こだわりポイント: 多くのゲームはデッドゾーン内を0、超えた瞬間にraw値をそのまま使うため閾値付近で入力がジャンプする。本作ではデッドゾーン超過後の値を (value - deadzone) / (1 - deadzone) で0〜1に再マッピングし、ジャンプを完全に排除。スティックのわずかな傾き始めから1:1でリニアに車が応答する。

スティック & トリガー閾値

STICK_DEADZONE = 0.12 // 左スティック
TRIGGER_DEADZONE = 0.08 // LT / RT

// 再マッピング: 閾値を超えた分を0〜1に正規化
if |value| < deadzone: return 0
return sign × (|value| - dz) / (1 - dz)

スティックは12%、トリガーは8%。トリガーはバネの戻りが強く微小入力が少ないため低めの閾値を採用。

KB / ゲームパッド自動切替

// ステアリング: 絶対値が大きい方を採用
kb = Input.GetAxis("Horizontal")
pad = ApplyDeadzone(leftStick.x)
steer = |pad| > |kb| ? pad : kb

// スロットル: KB(W/↑)とRT(右トリガー)の大きい方
// ※左スティックY軸は無視（コーナリング中の誤入力防止）

キーボードとゲームパッドの同時接続に対応。デバイス切替なしで自然に入力が引き継がれる。スロットルにスティックY軸を使わないのは、横に倒したときの斜め入力（コーナリングブリード）を防ぐため。

Source: locaCarcontroller.cs — デッドゾーン再マッピング 
private const float STICK_DEADZONE = 0.12f;
private const float TRIGGER_DEADZONE = 0.08f;

private static float ApplyDeadzone(float value, float deadzone)
{
    float abs = Mathf.Abs(value);
    if (abs < deadzone) return 0f;
    float sign = Mathf.Sign(value);
    return sign * (abs - deadzone) / (1f - deadzone);
}

🟄 動的ブレーキバイアス BRAKING

ステアリング入力に連動して前後ブレーキ配分をリアルタイム調整。コーナリング中の荷重移動を活かしたトレイルブレーキングを自然に実現。

こだわりポイント: 実車のレースカーはブレーキバイアスを手動調整するが、ゲームではプレイヤーがブレーキを踏みながらステアリングを切る「トレイルブレーキング」を自然にできることが重要。ステアリング量に応じてフロント寄り65%の配分を50:50へ自動シフトし、コーナー進入時のフロントロックを防止。さらにスムージング済みではなく生のステアリング入力を使用し、ブレーキ配分の応答を即座に反映する。

ステアリング連動バイアス

// 直進時: フロント65% / リア35%
// フルステア時: フロント50% / リア50%

steerRatio = |rawSteerInput|
frontBias = Lerp(0.65, 0.50, steerRatio)
rearBias = Lerp(0.35, 0.50, steerRatio)
brakeTorque = brakeInput × maxBrake × bias

直進ブレーキングではフロント荷重が増加するためフロント寄りに。コーナリング中はリアの荷重を活かすためイーブンに移行。

エンジンブレーキ＆ステアフェード

// エンジンブレーキ = エンジントルク × 0.015
// 駆動輪のみに適用

steerFade = 1 - |steerAngle/maxAngle| × 0.8
engineBrake = torque × 0.015 × gearRatio
             × finalDrive × steerFade

コーナリング中にエンジンブレーキが効きすぎると駆動輪がロックする。ステアリング角に応じて最大80%減衰させ、コーナー中のリアの安定性を確保。

⚙ シフトフィール SHIFT

シフトアップ時のトルクカットと速度ベースのオートシフトで、変速に「重み」と「賢さ」を持たせる。

こだわりポイント: RPMベースのオートシフトだと、タイヤが空転している時やコーナリング中に不適切なシフトアップが起きやすい。本作では各ギアの最高速度をタイヤ外径・ギア比・最大RPMから事前計算し、実車速が92%に達した時点でシフトアップする速度ベース方式を採用。さらにアップシフト時に0.12秒間のブレーキトルク（800Nm）を全輪に付与し、実車のシンクロメッシュのトルク途切れを再現。

Haptics

コントローラー振動

画面を見なくても路面状況がわかる — ダートの重い振動、縁石の鋭いパルス、タイヤのスリップバズ。4種類の振動が路面を「手で触れる」感覚に変える。

🔩 デュアルモーターアーキテクチャ CORE

低周波モーター（重い振動）と高周波モーター（細かいバズ）を独立制御。全トリガーの振動を加算合成し、毎フレームGamepad.current.SetMotorSpeeds(low, high)で出力する設計。

Dev Note: 各路面は振動の「質」が異なるため、プレイヤーはコントローラーを握っているだけで今どの路面を走っているか識別できる。ダートは低周波が支配的な重い揺れ、縁石は高周波パルスの鋭い刻み、アスファルトは無振動の静寂。さらにダート路面でタイヤがスリップすると、路面の重い低周波振動にスリップのバズが自然に重なり、単一トリガーでは表現できない複合的な触覚体験を加算式で実現。デッドゾーン処理で0.05未満の微振動をカットし、路面ごとのコントラストを際立たせている。

出力モデル

毎FixedUpdateで4トリガー（ダート・縁石・衝突・スリップ）それぞれのlowMotor/highMotor寄与を計算し、合算。最終値をMathf.Clamp01で0〜1に制限してからモーターへ出力する。各トリガーは独立した強度比率を持ち、低周波と高周波のバランスがトリガーごとに異なる。

デッドゾーン処理

合算後のlowMotor/highMotorが0.05未満の場合は0に切り捨て。微小な振動が常時発生してコントローラーが微震え続ける問題を防止。閾値はSerializeFieldで調整可能。

設定パラメータ

hapticsEnabledフラグで振動全体をON/OFF。各トリガーの強度（collisionRumbleScale, dirtRumbleLow/High, zebraRumble, gripRumbleIntensity）はInspectorから個別調整可能。タグ名（dirtTag, zebraTag）もSerializeFieldで変更可能。

ライフサイクル管理

OnDisable()でSetMotorSpeeds(0, 0)を呼び出し、車がDestroy/無効化されたときにモーターを確実に停止。ゲームパッド未接続時（Gamepad.current == null）は振動処理をスキップし、路面オーディオの更新だけを実行する。

🏟 ダート路面振動 SURFACE

WheelColliderの接地判定でダート路面を検出し、速度に比例した連続振動を生成。砂利道を走る重厚なフィードバック。

路面検出

4輪すべてのWheelCollider.GetGroundHit(out hit)を毎フレームチェック。hit.collider.CompareTag(dirtTag)が1輪でもtrueならisOnDirt = true。タグ名はデフォルト"Dirt"でSerializeFieldから変更可能。

速度スケーリング

Mathf.Clamp01(speedMS / 30f)で現在速度を0〜1に正規化。停車時は振動ゼロ、30 m/s（108 km/h）で最大強度。低速走行ではわずかな振動、速度が上がるにつれ砂利の粒が激しく跳ねる感覚を再現。

🚨 縁石パルス振動 SURFACE

コース端の縁石（ゼブラゾーン）を通過するときの等間隔バンプ感。Sin波パルスで振動周波数が速度に連動する。

実際のゲーム画面: コース端の赤白緑の縁石（ゼブラゾーン）。WheelColliderが「zebra」タグのコライダーを検出すると振動が発生する。

Dev Note: 実車の縁石は等間隔に配置された凸凹。低速で乗ると「ゴトゴト」、高速で乗ると「ブブブブ」と振動の質が変わる。この物理現象をSin(time * freq * PI)のパルス周波数変化で再現。

タイヤが等間隔の凸凹を踏むたびにSin波パルスが発生。速度が上がるほど周波数が増加し、振動の質が「ゴトゴト」→「ブブブブ」に変化。

路面検出

ダート同様、4輪のGetGroundHitでzebraTag（デフォルト"Zebra"）を判定。1輪でも縁石に乗っていればisOnZebra = true。

パルス周波数

Mathf.Lerp(8f, 45f, Clamp01(speedMS / 30f))で速度に応じて8Hz〜45Hzに変化。低速で8Hz（1秒に8回の振動）、高速で45Hz（1秒に45回）。人間が「個別のバンプ」として感じる限界付近の周波数帯を使用。

Sin波振動

Mathf.Abs(Mathf.Sin(Time.time * zebraFreq * Mathf.PI))で0〜1を繰り返すパルスを生成。Absで常に正の値にし、滑らかな「乗り上げ→離脱」サイクルを作る。速度スケールClamp01(speedMS / 20f)を掛けて低速時の振動を抑制。

モーター比率

低周波: zebraRumble * 0.5 * pulse * speedScale。高周波: zebraRumble * pulse * speedScale。高周波を低周波の2倍にすることで「硬い凸凹を叩く」鋭いバズ感を強調。zebraRumbleデフォルト0.8。

Source: locaCarcontroller.cs — 縁石パルス振動 
// Zebra/Kerb: speed-dependent pulsating vibration

{
    // パルス周波数: 低速8Hz → 高速45Hz (縁石の凸凹を踏む間隔)
    float zebraFreq = Mathf.Lerp(8f, 45f, Mathf.Clamp01(speedMS / 30f));

    highMotor += zebraRumble * pulse * speedScale;          // 鋭いバズ
}

💥 衝突インパクト COLLISION

OnCollisionEnterで壁やオブジェクトとの衝突を検知。衝撃力に比例した強烈な振動を一定時間かけて減衰させる。

衝撃力の算出

collision.impulse.magnitude / Mathf.Max(Time.fixedDeltaTime, 0.001f)で衝撃力を取得。物理エンジンが返すインパルス（力積）をフレーム時間で割って瞬間的な力に変換。車体質量（rb.mass、デフォルト1500kg）で正規化し、collisionRumbleScale（デフォルト0.8）を掛けてClamp01。

最低強度保証

Mathf.Max(intensity, 0.9f)で算出値が0.9未満でも90%に引き上げ。低速での軽い接触でもプレイヤーに「ぶつかった」と明確に伝わる設計。壁への擦り接触と正面衝突の強度差は残しつつ、最低ラインを保証。

線形減衰

collisionRumbleTimerをcollisionRumbleDuration（デフォルト0.4秒）から0へ毎フレーム減算。fade = Clamp01(timer / duration)を強度に掛けることで、衝突直後が最大→0.4秒後にゼロへ滑らかに減衰。低周波・高周波ともにintensity * fade * 1.0で均等に出力。

衝突の上書き判定

既に振動中に新たな衝突が発生した場合、intensity > collisionRumbleIntensity * (timer / duration)で「現在の残留振動より強いか」を比較。強い場合のみタイマーと強度を更新。連続衝突で弱い衝撃に上書きされることを防止。

void OnCollisionEnter(Collision collision)
{
    if (!hapticsEnabled) return;

    // インパルスをフレーム時間で割って瞬間的な力に変換
    float impactForce = collision.impulse.magnitude
        / Mathf.Max(Time.fixedDeltaTime, 0.001f);
    float intensity = Mathf.Clamp01(
        impactForce * collisionRumbleScale / (rb != null ? rb.mass : 1500f));
    intensity = Mathf.Max(intensity, 0.9f); // 最低90% - 必ず強く感じる

    // 現在の残留振動より強い場合のみ更新
    if (intensity > collisionRumbleIntensity
        * (collisionRumbleTimer / Mathf.Max(collisionRumbleDuration, 0.01f)))
    {
        collisionRumbleIntensity = intensity;
        collisionRumbleTimer = collisionRumbleDuration;
    }
}

💨 タイヤスリップ振動 GRIP

タイヤのグリップ使用率を監視し、限界を超えたスライド時にバズ寄りの振動フィードバック。ドリフトやロックアップを手で感じ取れる。

グリップ使用率の算出

各輪のWheelHitから前後スリップforwardSlipと横スリップsidewaysSlipを取得。それぞれを摩擦カーブのasymptoteSlipで正規化し、4輪中の最大値をmaxGripUsageとして採用。1.0が摩擦曲線のピーク、それ以上はグリップ喪失。

スムージング

currentGripUsage = Mathf.Lerp(currentGripUsage, maxGripUsage, Time.fixedDeltaTime * 12f)で急激な変動を抑制。12倍のレートで約0.08秒の時定数。タイヤが瞬間的にスリップしても振動がチカチカせず、グリップ回復も滑らかにフェード。値はClampせず1.0超の極端なスライドも追従。

閾値判定

skidGripThreshold（デフォルト5.0）を超えたときのみ振動発生。InverseLerp(skidGripThreshold, 4.0, currentGripUsage)で閾値からのフェードインを算出。通常走行では絶対に振動せず、激しいスライドでのみ作動する設計。閾値が高いほどトリガーが厳しくなる。

モーター比率

低周波: gripRumbleIntensity * gripFade * 0.5。高周波: gripRumbleIntensity * gripFade * 0.8。gripRumbleIntensityデフォルト0.4。高周波が低周波の1.6倍で、タイヤが路面を「擦る」ザラついたバズ感を強調。ダートの重い振動とは対照的な質感。
