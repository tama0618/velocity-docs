🔧 エンジン＆トランスミッション CORE

RPM（回転数）ベースのエンジンモデル。トルクカーブ、ギア比、ファイナルドライブまで実車に準拠した計算。

こだわりポイント: AnimationCurveで定義したトルクカーブは5つのキーポイントで構成。低回転域ではトルクが立ち上がり、中回転でピーク、高回転で自然に落ち込む実車特性を再現。プレイヤーはパワーバンドを意識したシフト操作が求められます。

トルクカーブ (AnimationCurve)

5点のキーフレームで定義。RPM正規化値(0-1)に対するトルク倍率:

  • 0% (0 RPM) → 倍率 30%
  • 35% (2,450 RPM) → 倍率 80%
  • 55% (3,850 RPM) → 倍率 100% (ピーク)
  • 80% (5,600 RPM) → 倍率 95%
  • 100% (7,000 RPM) → 倍率 70%
driveTorque = maxTorque × curve(rpm/maxRPM)
    × gearRatio × finalDrive × throttle
wheelTorque = driveTorque / wheelRadius

5速ギアボックス

  • ギア比: 3.5 / 2.3 / 1.7 / 1.3 / 1.0
  • ファイナルドライブ: 3.8
  • 速度域: 65 / 100 / 135 / 176 / 230 km/h
  • シフトクールダウン: 0.3秒
rpm = wheelRPM × gearRatio × finalDrive
rpm = Clamp(rpm, idleRPM, maxRPM)
// wheelRPM = speed / (2π × radius) × 60

AT/MT切替

ワンボタンでオートマ/マニュアル切替。ATモードはRPMに基づく自動変速:

// シフトダウン判定
rpm < 3000 AND gear > 1

🛒 タイヤ摩擦モデル PHYSICS

WheelColliderのFrictionCurveを詳細設定。前輪と後輪で異なる特性を持たせ、実車のようなアンダー/オーバーステア特性を再現。

こだわりポイント: 前輪は横グリップを高めに設定しレスポンスの良い初期旋回を実現。後輪はstiffnessを1.25倍に設定することで、限界域では後輪が先に滑り出すリアルなオーバーステア特性を再現。この前後差がドライビングの「手応え」を生みます。

前輪フリクション

  • 前方: extremumSlip=0.14, value=1.15
  • 前方: asymptoteSlip=0.35, value=0.95
  • 横方: extremumSlip=0.22, value=1.45
  • 横方: asymptoteSlip=0.55, value=1.30
  • 横方stiffness: 1.0

後輪フリクション

  • 前方: 前輪と同一カーブ
  • 横方: 前輪と同一カーブ
  • 横方stiffness: 1.25 (後輪を硬めに設定)

後輪の横stiffnessを25%増すことで、高荷重時に後輪が安定し、低荷重時（ブレーキング等）に自然なオーバーステアを発生。

サスペンション

  • ストローク: 0.22 m
  • スプリング: 42,000 N/m
  • ダンパー: 10,000 N·s/m
  • ターゲットポジション: 0.5 (中間)

重心 & 空力

  • 重心オフセット: (0, -0.45, 0.05)
  • ダウンフォース係数: 1.8

🛡 安全制御システム ASSIST

3つの独立した車両安定化システム。各システムは個別にON/OFF可能。

こだわりポイント: ESCは実車同様「理想ヨーレート」と「実際のヨーレート」の差分を監視。ホイールベースと速度から物理的に正しい理想値を計算し、オーバーステア/アンダーステアそれぞれに異なるブレーキ制御を適用。OFFにすればドリフト走行も可能です。

ABS（アンチロックブレーキ）

スリップ閾値 0.06 を超えるとブレーキ圧を15Hzでパルス制御。

slip = (wheelSpeed - groundSpeed) / max(groundSpeed)
if |slip| > 0.06:
  brake = 0 // パルスOFF
// 15Hz周期でON/OFF切替

TCS（トラクションコントロール）

駆動輪のスリップが 0.12 を超えるとモータートルクを低減。

if driveSlip > 0.12:
  motorTorque *= 0.3
// ステアリング角で閾値を動的拡大
threshold += |steerAngle| × 0.02

ESC（横滑り防止）

idealYaw = steerAngle × speed / wheelBase
error = actualYaw - idealYaw
deadzone = 0.2 rad/s

// オーバーステア: 外側前輪にブレーキ
// アンダーステア: 内側後輪にブレーキ
correction = maxBrake × 0.30 × |error|

アンチロールバー

antiRoll = (travelL - travelR) × force
Front: force = 5,500 N·m/rad
Rear: force = 3,500 N·m/rad

左右のサスペンション伸縮差からロール対抗力を算出。コーナリング時の車体傾斜を抑制。

🎯 ステアリング＆ゴーストモード

速度感応式ステアリングとスピン時の安全システム。

こだわりポイント: ステアリング角度は低速で広く、高速で狭くなる速度感応式。さらに高速域ではスムージングを6倍に強化し、誤操作による急激な挙動変化を防止。パッシブヨーダンピングで実車のセルフアライニングトルクを模擬しています。

速度感応ステアリング

// 高速スムージング

  smoothing ×= 6.0

// ヨーダンピング

👁 ルックアヘッド（先読み）システム LOOK-AHEAD

速度・コース逸脱度・カーブ曲率の3要素でルックアヘッド距離を動的に決定。

lookAheadSec = Lerp(0.3, 0.4, speedKmh / 180)
↓ コース逸脱時
lookAheadSec ×= Clamp01(1 - (deviation - 1.5) / 8) + 0.25
↓ オフトラック時
lookAheadSec ×= 0.4
↓ ポイント数に変換
lookAheadPts = Max(3, Round(lookAheadSec / 0.05))

カーブ曲率検出アルゴリズム

ルックアヘッド範囲内を2ポイントおきにサンプリングし、隣接セグメント間の角度変化を計測。

maxAngleChange > 20° のとき:
curveFactor = Lerp(1.0, 0.5, Clamp01((maxAngle - 20) / 40))
lookAheadPts = Max(3, Round(lookAheadPts × curveFactor))

20°で100% → 40°で75% → 60°以上で50%まで短縮。S字コーナーやシケインの事前検知に使用。

逸脱時の挙動変化

  • deviation > 1.5m: ルックアヘッド短縮（ライン復帰優先）
  • isOffTrack: ルックアヘッド60%カット（0.4倍）
  • deviation > 4m: 速度ペナルティ発動
  • deviation > 30m: テレポートリカバリー

🛠 ステアリングアルゴリズム STEERING

角度正規化 → 感度ブースト → ノイズ注入 → ハイブリッドスムージングの4段パイプライン。

Step 1: 角度正規化

angleToTarget = SignedAngle(forward, toTarget, up)
sens = Lerp(1.5, 1.6, speedRatio)
normalized = Clamp(angleToTarget / 45 × sens, -1, 1)

45°で±1.0に到達する設計。90°除算だとフルロックに達しづらいため、45°基準を採用。

Step 2: 感度ブースト

コース逸脱度に応じて感度を最大3倍まで引き上げ。

deviation > 1.5m:
  sens ×= Lerp(1, 3, Clamp01((dev-1.5)/5))
isOffTrack:
  sens ×= 1.8

Step 3: Perlinノイズ注入

AIの走行を機械的にしないための微小なステアリング揺らぎ。オフトラック時は無効化。

noise = (PerlinNoise(time×2, offset) × 2 - 1)
       × 0.005
normalized += noise

Step 4: ハイブリッドスムージング

Lerp（指数減衰型）とMoveTowards（定速型）を毎フレーム並行計算し、目標値に近い方を採用。小修正はLerpが有利、大角度修正はMoveTowardsが有利。

smoothing = 12
// 逸脱時: ×3, オフトラック: ×2.5, S字切返し: ×2

lerped = Lerp(current, target, dt × smoothing)
moved = MoveTowards(current, target, dt × smoothing × 0.5)

current = |target - moved| < |target - lerped| ? moved : lerped

🛑 ブレーキング制御 BRAKING

先読みベースの予測ブレーキ + ステアリング連動減速 + コーナーブースト低減の3層制御。

予測ブレーキアルゴリズム

// 先読みポイント数（速度依存）
brakeLookPts = Round(Lerp(20, 50, speedRatio))
// レーススタート時は+25ポイント延長

// 将来の最低速度を検索
lowestFutureSpeed = Min(points[i].speed × 1.05) for i in range

// ブレーキ閾値判定
if lowestFuture < targetSpeed × 0.88:
  targetSpeed = Lerp(lowestFuture, targetSpeed, 0.25)

レーススタート時（最初の10秒）は閾値0.92、ブレンド率0.2に変更し、より早くブレーキを開始。

ステアリング連動減速

if |steering| > 0.5:
  slowdown = Lerp(1.0, 0.9,
    Clamp01((|steer|-0.5)/0.4))
  targetSpeed ×= slowdown

ステアリング0.5以上で最大10%減速。

コーナー前ブースト低減

futureDropRatio = lowestFuture / baseSpeed

if dropRatio < 0.9:
  reduce = InverseLerp(0.9, 0.5, drop)
  speedBoost = Lerp(boost, 1.0, reduce)

コーナー手前で速度が10%以上落ちる場合、順位ブーストを段階的に解除。

🚗 車線変更 & 回避システム TACTICS

5つのレーン候補をスコアリングし、最適な走行ラインをリアルタイムで選択。壁レイキャストで安全範囲を制限。

5レーン候補

  • CENTER: offset = 0
  • FULL RIGHT: offset = +3.5m
  • FULL LEFT: offset = -3.5m
  • HALF RIGHT: offset = +1.75m
  • HALF LEFT: offset = -1.75m

各候補は壁レイキャスト（現在地＋20pt先＋40pt先）で安全範囲にクランプ。検知範囲: 前方15m。

スコアリング公式

score = minSeparation × 3.0
   + (1 - |cand - current| / range) × 1.5
   + (1 - |cand| / maxOffset) × 0.8
   + sameSideBonus × 3.0
   + oppositeSideBonus × 2.5

他車距離 (x3)が最重要。次にレーン維持安定性(x1.5)、理想ライン近接度(x0.8)、同一方向ボーナス(x3)。

壁検出 & 壁回避

detectDist = Lerp(5, 20, speedKmh/150)
// 前方 + ±30°の3方向レイキャスト

proximity = 1 - hitDist / detectDist
wallBrake = proximity × 0.5
wallSteer = sign(normal·right) × proximity × 0.6

テールゲート防止

前走車に2-3秒以上接近し続けるとタイマー発動。最大40%の速度低減を段階的に適用。

wallSqueeze: speed ×= Lerp(1.0, 0.8, squeeze)
tailgate: speed ×= Lerp(1.0, 0.75, tailgateFactor)

📌 レーシングライン生成 SPLINE

ウェイポイントをCatmull-Romスプラインで補間し、曲率ベースのオフセットで理想ラインを計算。Laplacianスムージングで最終形状を整形。

Catmull-Romスプライン公式

P(t) = 0.5 × [
  (2×P1) +
  (-P0 + P2) × t +
  (2P0 - 5P1 + 4P2 - P3) × t² +
  (-P0 + 3P1 - 3P2 + P3) × t³
]

4つの制御点(P0-P3)とパラメータt(0-1)から滑らかな曲線上の点を算出。splineResolution=10で各ウェイポイント間を10分割。

曲率ベースのオフセット

curvature = Cross(dirPrev, dirNext).y
strength = Clamp(|curv| × 10 × 8, 0, 1)
offset = sign(curv) × strength
      × roadWidth × 0.4 × 0.6

曲率が大きいほどコーナー外側にオフセット。道幅の24%まで使用。

3パスLaplacianスムージング

// 3回繰り返し
for pass in 0..2:
  smoothed[i] = points[i] × 0.5
   + points[i-1] × 0.25
   + points[i+1] × 0.25

ルックアヘッド距離

lookAheadDist = Lerp(8, 40, InverseLerp(20, 150, speedKmh))

20km/hで8m先、150km/hで40m先を参照。速度に応じて線形補間。

🛠 ステアリング制御 STEERING

非線形パワーカーブ + 速度依存スムージング + オーバーテイクバイアスの3段階制御。

非線形ステアリングカーブ

normalized = angleToWP / 90
curvedValue = Pow(|normalized|, 0.7)
targetSteering = Clamp(sign × curvedValue × 1.0, -1, 1)

指数0.7のパワーカーブで中間域を穏やかに、大角度域をシャープに。90°で±1.0に到達。

速度依存スムージング

speedFactor = Lerp(1.5, 0.6,
  InverseLerp(30, 150, speedKmh))
angleDiff = |target - current|
smoothFactor = Lerp(1.0, 0.4, angleDiff)
lerpSpeed = 5 × speedFactor × smoothFactor

current = Lerp(current, target, dt × lerpSpeed)

低速(30km/h)では素早く(1.5倍)、高速(150km/h)では慎重に(0.6倍)。

オーバーテイクバイアス

if |angle| < 20°:
  angleFactor = 1 - (|angle| / 20)
  current += overtakeBias × angleFactor

// 回避バイアス公式
distFactor = InverseLerp(12, 1, aiDist)
curveFactor = Lerp(1, 0.3,
  InverseLerp(15, 40, |angle|))
speedUrg = Lerp(1.5, 0.8, speed/50)
bias = dir × 0.35 × distF × curveF
   × speedUrg × wallDampen

💨 速度制御 & ブレーキング

物理ベースの最適速度計算 + 運動方程式ブレーキ距離 + 角度別速度制限。

物理ベース最適速度

vmax = √(μ × g × r) × 3.6 × safetyMargin
// μ = gripFactor(2.2) × 0.8 = 1.76
// g = 9.81, r = コーナー半径

ブレーキ距離逆算

// 運動方程式: v² = v0² + 2ad
v0_max = √(v_next² + 2 × decel × dist)

// カーブ角度で減速率を補正
>60°: effectiveDecel ×= 0.7
>40°: effectiveDecel ×= 0.85

角度別速度制限

角度速度上限
> 90°30 km/h
> 60°45 km/h
> 40°60 km/h
> 10°Lerp(100%, 50%)

TTC (衝突予測時間) ブレーキ

ttc = distance / closingSpeed

ttc < 0.5s: フルブレーキ
ttc < 1.5s: 強度 = InverseLerp(1.5, 0.5, ttc)
ttc < 3.0s: 軽減速

スリップストリーム

// 5-20m後方 AND 横2m以内
targetSpeed ×= 1.05 (5%ブースト)
lingering = 3秒持続

🚦 障害物回避 & リカバリー AVOIDANCE

レイキャスト構成

  • 前方検出: 5方向 (0°, ±10°, ±20°), 25m
  • 障害物検出: 5方向 (0°, ±20°, ±40°), 4m
  • 壁回避: 横方向, 3m
  • リバース確認: 前後5方向, 前3.5m/後2.5m
  • 路幅計測: 左右1mステップ, 最大15m
  • 地面検出: 下方5m

回避方向決定

waypointBias = sign(angleToWP) × 0.3
random = Range(-1, 1) + waypointBias

if |aiCarRelPos.x| > 0.5:
  random -= sign(relPos.x) × 0.8

direction = (random ≥ 0) ? +1 : -1

スタック検出 & リカバリー

// 条件1: 低速 + スロットル + 障害物
speed < 3km/h AND accel > 0.3 AND obstacle
// 条件2: ほぼ停止 + スロットル
speed < 1km/h AND accel > 0.5

reverseTime = 2.0 + Min(stuckCount×0.5, 2.5)

Systems

レース管理システム

カウントダウン、順位管理、ラップ計測を統合的に制御

⏱ カウントダウン

RaceCountdown.cs

画面中央に大文字で 3 → 2 → 1 → GO! を表示。GO!の1秒後にフェードアウト。カウント中は全車両の canMove = false で停止。完了時に EnableMovement() を全車に送信。

🏆 順位管理

RacePositionManager.cs

0.2秒間隔で全車の進行度を計算。進行度 = ラップ数 + (ポイントインデックス / 総ポイント数)。プレイヤーは最寄りポイント検索で位置を推定。順位に応じてAIのブーストファクターを自動更新。

⏲ ラップタイマー

GoalManager.cs

トリガーベースのラップ検出（重複防止: 50秒間隔）。初回通過でタイマー開始、2回目以降でラップタイム記録。現在ラップ・前ラップ・ベストラップをTMPで表示。表示形式: MM:SS.CC。

💾 ラップ記録

LapRecorder.cs

0.05秒間隔(20Hz)で車両状態を記録。位置・回転・速度・ステア角・トルク値を保存。ゴール通過で自動分割。JSON形式でDesktop/lap_dataフォルダに出力。最低10ポイント以上で保存。
