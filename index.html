🔧 エンジン＆トランスミッション CORE

RPM（回転数）ベースのエンジンモデル。トルクカーブ、ギア比、ファイナルドライブまで実車に準拠した計算。

こだわりポイント: AnimationCurveで定義したトルクカーブは5つのキーポイントで構成。低回転域ではトルクが立ち上がり、中回転でピーク、高回転で自然に落ち込む実車特性を再現。プレイヤーはパワーバンドを意識したシフト操作が求められます。

トルクカーブ (AnimationCurve)

5点のキーフレームで定義。RPM正規化値(0-1)に対するトルク倍率:

  • 0% (0 RPM) → 倍率 30%
  • 35% (2,450 RPM) → 倍率 80%
  • 55% (3,850 RPM) → 倍率 100% (ピーク)
  • 80% (5,600 RPM) → 倍率 95%
  • 100% (7,000 RPM) → 倍率 70%
driveTorque = maxTorque × curve(rpm/maxRPM)
    × gearRatio × finalDrive × throttle
wheelTorque = driveTorque / wheelRadius

5速ギアボックス

  • ギア比: 3.5 / 2.3 / 1.7 / 1.3 / 1.0
  • ファイナルドライブ: 3.8
  • 速度域: 65 / 100 / 135 / 176 / 230 km/h
  • シフトクールダウン: 0.3秒
rpm = wheelRPM × gearRatio × finalDrive
rpm = Clamp(rpm, idleRPM, maxRPM)
// wheelRPM = speed / (2π × radius) × 60

AT/MT切替

ワンボタンでオートマ/マニュアル切替。ATモードはRPMに基づく自動変速:

// シフトアップ判定
rpm > 6500 AND speed > gearLimit[gear] × 0.9
// シフトダウン判定
rpm < 3000 AND gear > 1

エンジンブレーキ & シンクロ

スロットルOFF時、最大トルクの1.5%がエンジンブレーキとして作用。

engineBrake = maxTorque × 0.015 × gearRatio
// ギアチェンジ時
syncResistance = 800 N·m for 0.12秒

🛒 タイヤ摩擦モデル PHYSICS

WheelColliderのFrictionCurveを詳細設定。前輪と後輪で異なる特性を持たせ、実車のようなアンダー/オーバーステア特性を再現。

こだわりポイント: 前輪は横グリップを高めに設定しレスポンスの良い初期旋回を実現。後輪はstiffnessを1.25倍に設定することで、限界域では後輪が先に滑り出すリアルなオーバーステア特性を再現。この前後差がドライビングの「手応え」を生みます。

前輪フリクション

  • 前方: extremumSlip=0.14, value=1.15
  • 前方: asymptoteSlip=0.35, value=0.95
  • 横方: extremumSlip=0.22, value=1.45
  • 横方: asymptoteSlip=0.55, value=1.30
  • 横方stiffness: 1.0

後輪フリクション

  • 前方: 前輪と同一カーブ
  • 横方: 前輪と同一カーブ
  • 横方stiffness: 1.25 (後輪を硬めに設定)

後輪の横stiffnessを25%増すことで、高荷重時に後輪が安定し、低荷重時（ブレーキング等）に自然なオーバーステアを発生。

サスペンション

  • ストローク: 0.22 m
  • スプリング: 42,000 N/m
  • ダンパー: 10,000 N·s/m
  • ターゲットポジション: 0.5 (中間)

重心 & 空力

  • 重心オフセット: (0, -0.45, 0.05)
  • ダウンフォース係数: 1.8

🛡 安全制御システム ASSIST

3つの独立した車両安定化システム。各システムは個別にON/OFF可能。

こだわりポイント: ESCは実車同様「理想ヨーレート」と「実際のヨーレート」の差分を監視。ホイールベースと速度から物理的に正しい理想値を計算し、オーバーステア/アンダーステアそれぞれに異なるブレーキ制御を適用。OFFにすればドリフト走行も可能です。

ABS（アンチロックブレーキ）

スリップ閾値 0.06 を超えるとブレーキ圧を15Hzでパルス制御。

slip = (wheelSpeed - groundSpeed) / max(groundSpeed)
if |slip| > 0.06:
  brake = 0 // パルスOFF
// 15Hz周期でON/OFF切替

TCS（トラクションコントロール）

駆動輪のスリップが 0.12 を超えるとモータートルクを低減。

if driveSlip > 0.12:
  motorTorque *= 0.3
// ステアリング角で閾値を動的拡大
threshold += |steerAngle| × 0.02

ESC（横滑り防止）

idealYaw = steerAngle × speed / wheelBase
error = actualYaw - idealYaw
deadzone = 0.2 rad/s

// オーバーステア: 外側前輪にブレーキ
// アンダーステア: 内側後輪にブレーキ
correction = maxBrake × 0.30 × |error|

アンチロールバー

antiRoll = (travelL - travelR) × force
Front: force = 5,500 N·m/rad
Rear: force = 3,500 N·m/rad

左右のサスペンション伸縮差からロール対抗力を算出。コーナリング時の車体傾斜を抑制。

🎯 ステアリング＆ゴーストモード

速度感応式ステアリングとスピン時の安全システム。

こだわりポイント: ステアリング角度は低速で広く、高速で狭くなる速度感応式。さらに高速域ではスムージングを6倍に強化し、誤操作による急激な挙動変化を防止。パッシブヨーダンピングで実車のセルフアライニングトルクを模擬しています。

速度感応ステアリング

// 高速スムージング
if speed > 80km/h:
  smoothing ×= 6.0

// ヨーダンピング

ゴーストモード

⚙ 高度タイヤ物理 & 触覚フィードバック ADVANCED

タイヤリラクゼーション、トレイルブレーキング、トルクコンバーター、プロシージャル路面音、ゲームパッド触覚。locaCarcontroller v29の先進物理。

レースゲーム開発者へ: タイヤリラクゼーションとは、タイヤが滑り始めてから最大グリップに達するまでの遅延を模擬するモデル。実車のタイヤはゴムの弾性変形に時間がかかるため、ステアリングを切った瞬間にはグリップが発生せず、数十ミリ秒後にピークに達する。これがないと操作がデジタル的で、「タイヤが路面を掴む感覚」が失われる。
参考文献: "Race Car Vehicle Dynamics" - Milliken & Milliken, SAE International, Chapter 2: Tire Behavior

タイヤリラクゼーション (指数スムージング)

// 速度依存の追従レート
rate = Max(speedMS / 0.5, 14)
lerpFactor = 1 - exp(-rate × dt)

// 目標stiffnessへの指数減衰追従
smoothedStiffness = Lerp(current, target, lerpFactor)

// 低速(0m/s)では14Hz, 高速(28m/s)ではl56Hz
// → 高速ほど即座にグリップ変化が反映

トレイルブレーキング

// ブレーキ中にステアすると前輪横グリップ増加
frontSideGrip ×= 1 + 0.35 × brake × |steer|

// コーナリング中の前後バランス制御
// 前輪: ステア中は横グリップUP
frontSide ×= Lerp(1, Lerp(1.35,1.05,speedBlend), |steer|)
// 後輪: 低速で前方グリップUP, 高速で横グリップUP
rearSide ×= Lerp(1, Lerp(1.0,1.4,speedBlend), |steer|)
rearFwd ×= Lerp(1, Lerp(1.4,1.0,speedBlend), |steer|)

実車でブレーキングしながらターンインすると前輪荷重が増え、グリップが向上する現象を再現。レーシングドライバーの基本テクニック「トレイルブレーキング」を物理的にサポート。

プロシージャル路面音

  • ダート: LPF白色ノイズ + ランダムな砂利クランチスパイク。スリップ量に比例した音量。
  • ゼブラ (縁石): 30バンプ/秒のパルス列。8ms二次関数減衰の衝撃音。速度に比例したピッチ。
  • スキッド: AudioClipベース。グリップ使用率に比例した音量。フリクションサークルの飽和度で強度計算。

ゲームパッド触覚 (Haptics)

  • ダート走行: 低周波の連続ランブル。速度に比例。
  • ゼブラ通過: 8-45Hzのパルス振動。縁石のバンプを直接伝達。
  • 衝突: 衝撃比例の減衰振動。InverseLerpで0.2-1.0にマッピング。
  • グリップ使用率: タイヤの複合スリップ量に比例。限界域でコントローラーが振動し、限界を手で感じ取れる。

🔧 CarBodyKit - 車種カスタマイズシステム MODULAR

CarBodyKit.cs - C#リフレクションでlocalCarcontrollerの全パラメータを上書きし、1つの物理コントローラーで複数車種を表現するアーキテクチャ。ボディのビジュアル差し替えとphysics tuningを同時に行う。

レースゲーム開発者へ: 車種ごとに物理コントローラーを複製するのではなく、1つのコントローラー + 差分パラメータ方式を採用。これにより、新しい車種の追加はScriptableObjectやMonoBehaviourで差分だけ定義すればよく、物理エンジンのバグ修正が全車種に一括反映される利点があります。
参考文献: "Game Programming Patterns" - Robert Nystrom, Component Pattern / Type Object Pattern

アーキテクチャ: Reflection Override

C#のSystem.Reflectionでprivateフィールドを直接書き換え。[DefaultExecutionOrder(10)]でAwake実行順を制御し、localCarcontrollerのStart()より先にパラメータを注入。

// 実行順序の制御
[DefaultExecutionOrder(10)] // controller(0)より後
[RequireComponent(typeof(localCarcontroller))]

// Awakeで全パラメータを上書き
OverrideField("tirePhysicsVersion", 999);
// ↑ controllerのStart()内リセットを回避

// リフレクションでprivateフィールド書き換え
var field = typeof(localCarcontroller)
  .GetField(name, NonPublic | Instance);
field.SetValue(controller, value);

👁 ルックアヘッド（先読み）システム LOOK-AHEAD

速度・コース逸脱度・カーブ曲率の3要素でルックアヘッド距離を動的に決定。

lookAheadSec = Lerp(0.3, 0.4, speedKmh / 180)
↓ コース逸脱時
lookAheadSec ×= Clamp01(1 - (deviation - 1.5) / 8) + 0.25
↓ オフトラック時
lookAheadSec ×= 0.4
↓ ポイント数に変換
lookAheadPts = Max(3, Round(lookAheadSec / 0.05))

カーブ曲率検出アルゴリズム

ルックアヘッド範囲内を2ポイントおきにサンプリングし、隣接セグメント間の角度変化を計測。

maxAngleChange > 20° のとき:
curveFactor = Lerp(1.0, 0.5, Clamp01((maxAngle - 20) / 40))
lookAheadPts = Max(3, Round(lookAheadPts × curveFactor))

20°で100% → 40°で75% → 60°以上で50%まで短縮。S字コーナーやシケインの事前検知に使用。

逸脱時の挙動変化

  • deviation > 1.5m: ルックアヘッド短縮（ライン復帰優先）
  • isOffTrack: ルックアヘッド60%カット（0.4倍）
  • deviation > 4m: 速度ペナルティ発動
  • deviation > 30m: テレポートリカバリー

🛠 ステアリングアルゴリズム STEERING

角度正規化 → 感度ブースト → ノイズ注入 → ハイブリッドスムージングの4段パイプライン。

Step 1: 角度正規化

angleToTarget = SignedAngle(forward, toTarget, up)
sens = Lerp(1.5, 1.6, speedRatio)
normalized = Clamp(angleToTarget / 45 × sens, -1, 1)

45°で±1.0に到達する設計。90°除算だとフルロックに達しづらいため、45°基準を採用。

Step 2: 感度ブースト

コース逸脱度に応じて感度を最大3倍まで引き上げ。

deviation > 1.5m:
  sens ×= Lerp(1, 3, Clamp01((dev-1.5)/5))
isOffTrack:
  sens ×= 1.8

Step 3: Perlinノイズ注入

AIの走行を機械的にしないための微小なステアリング揺らぎ。オフトラック時は無効化。

noise = (PerlinNoise(time×2, offset) × 2 - 1)
       × 0.005
normalized += noise

Step 4: ハイブリッドスムージング

Lerp（指数減衰型）とMoveTowards（定速型）を毎フレーム並行計算し、目標値に近い方を採用。小修正はLerpが有利、大角度修正はMoveTowardsが有利。

smoothing = 12
// 逸脱時: ×3, オフトラック: ×2.5, S字切返し: ×2

lerped = Lerp(current, target, dt × smoothing)
moved = MoveTowards(current, target, dt × smoothing × 0.5)

current = |target - moved| < |target - lerped| ? moved : lerped

🛑 ブレーキング制御 BRAKING

先読みベースの予測ブレーキ + ステアリング連動減速 + コーナーブースト低減の3層制御。

予測ブレーキアルゴリズム

// 先読みポイント数（速度依存）

// レーススタート時は+25ポイント延長

// 将来の最低速度を検索
lowestFutureSpeed = Min(points[i].speed × 1.05) for i in range

📌 レーシングライン生成 SPLINE

ウェイポイントをCatmull-Romスプラインで補間し、曲率ベースのオフセットで理想ラインを計算。Laplacianスムージングで最終形状を整形。

Catmull-Romスプライン公式

P(t) = 0.5 × [
  (2×P1) +
  (-P0 + P2) × t +
  (2P0 - 5P1 + 4P2 - P3) × t² +
  (-P0 + 3P1 - 3P2 + P3) × t³
]

4つの制御点(P0-P3)とパラメータt(0-1)から滑らかな曲線上の点を算出。splineResolution=10で各ウェイポイント間を10分割。

曲率ベースのオフセット

curvature = Cross(dirPrev, dirNext).y
strength = Clamp(|curv| × 10 × 8, 0, 1)
offset = sign(curv) × strength
      × roadWidth × 0.4 × 0.6

曲率が大きいほどコーナー外側にオフセット。道幅の24%まで使用。

3パスLaplacianスムージング

// 3回繰り返し
for pass in 0..2:
  smoothed[i] = points[i] × 0.5
   + points[i-1] × 0.25
   + points[i+1] × 0.25

🧠 適応学習 & パーソナリティ ADAPTIVE

走行中にコーナー速度を自動調整する適応学習と、AI車両ごとに異なる運転スタイルを生成するパーソナリティシステム。

レースゲーム開発者へ: 固定の速度マップだけだとAIは同じコーナーで何度もスピンする。適応学習は各ウェイポイントでスリップ量を監視し、次回通過時の速度係数を自動で下げる。これにより、AIはコースを「覚える」。パーソナリティは同じAIロジックでもバリエーションを生み出す簡単な手法。
参考: "AI Game Programming Wisdom" - Steve Rabin, Chapter: Adaptive AI

  speedModifier[wp] -= learningRate(0.05)
  speedModifier[wp] = Max(0.6, modifier)
// 最低60%まで減速

// 安定走行時の回復
if stable:
  speedModifier[wp] += learningRate × 0.1
// 学習速度の10%で回復 → 慎重に速度を戻す

パーソナリティ (ランダム分散)

gripFactor ×= Random(0.85, 1.15)
overtakeAggression ×= Random(0.85, 1.15)

コーナー半径計算 (ヘロンの公式)

// 3ウェイポイント(三角形)から外接円の半径を求める
a = |P1 - P0|, b = |P2 - P1|, c = |P0 - P2|
s = (a + b + c) / 2 // 半周長
A = √(s × (s-a) × (s-b) × (s-c)) // ヘロン
R = a × b × c / (4 × A) // 外接円半径

// この半径を物理ベース最適速度に使用:
vmax = √(μ × g × R) × 3.6 × safetyMargin

3点の座標だけからコーナーの曲率半径を正確に算出。ウェイポイント間隔が不均等でも正しく動作するため、手動配置のウェイポイントに対してロバスト。

numPeriods = sampleRate / periodSamples
// 偶数に丸め (ループポイント整合)
totalSamples = periodSamples × numPeriods

// 2. ブロック共振 (減衰振動)

  × exp(-phase × decay × 0.7) × amount

// 3. サブベース (クランク回転周波数)

// 4. 燃焼ノイズ (パルスエンベロープ連動)
noise = LPF(white_noise, cutoff)
  × exp(-phase × decay × 0.4) × amount

  vtecTarget = Clamp01((speed - 0.6) / 0.08)
  // 8%の幅で0→1に急変（バッと切り替わる感じ）

vtecBlend = Lerp(vtecBlend, target, dt × 20)
// 追従速度20 = 0.05秒で完了

// 音量クロスフェード
lowCamVol = rpmVolume × Lerp(1, 0.15, vtecBlend)
highCamVol = rpmVolume × vtecBlend
// lowCamRetain=0.15で完全には消さない

レブリミッター (燃料カット方式)

// normalizedSpeed ≥ 0.99 で発動
timer += dt × 12Hz × 2π
sinVal = sin(timer)
volumeMod = sinVal > 0 ? 1.0 : 0.3
// 矩形波ON/OFFで「バババ」音を再現
// ピッチは変えない (実車同様RPMは維持)

ピッチ計算 (locaCarcontrollerと同一アルゴリズム)

// リフレクションで現在ギアと速度上限を取得
normalizedSpeed = speedKmh / calculatedSpeedLimits[gear]
basePitch = Lerp(0.4, 1.9, normalizedSpeed)

// スムージング (急激なピッチ変化を防止)
source.pitch = Lerp(current, basePitch, dt × 10)

ギアごとに速度を正規化するため、シフトアップ時にピッチが自然に落ちて再び上昇する、実車と同じ挙動が得られる。

🔨 EJ257 ボクサーエンジン + ターボサウンド BOXER

CarBodyKit.cs内蔵 - 水平対向4気筒特有の不等間隔排気パルス + ターボホイッスル + ブローオフバルブの3レイヤー合成。

レースゲーム開発者へ: ボクサーエンジンの「ドロドロ」音の正体は不等間隔排気。通常の直4は等間隔点火(180°/180°/180°/180°)だが、水平対向では排気系統のレイアウトにより、聴覚上のパルス間隔が不均等になる。unevenRatio = 0.38（38:62比）でこれを模擬。

Layer 1: ボクサーランブル

// 点火周波数107Hz, 不等間隔比0.38
phase = (t × 107) % 1.0

// パルス1: 主点火
d1 = phase
if d1 < 0.35:
  env = exp(-d1 × 20)
  p1 = env × (sin(12πd) × 1.0
   + sin(4πd) × 0.8 + sin(8πd) × 0.4)

// パルス2: 不等間隔パルス
d2 = (phase - 0.38 + 1) % 1
// 同じ波形、位相を38%オフセット

raw = p1 + p2
output = LPF(raw, 0.28) + mech_noise × 0.05

Layer 2: ターボホイッスル

// 基本周波数3500Hz + 2倍音7000Hz
whine = sin(t × 3500 × 2π) × 0.65
harm = sin(t × 7000 × 2π) × 0.25
// 47Hz振幅変調(コンプレッサー回転揺らぎ)
mod = 1 + sin(t × 47 × 2π) × 0.25

// ブースト連動
targetBoost = throttle × Clamp01((rpm-0.2)/0.3)
volume = turboWhineVol × currentBoost
pitch = Lerp(0.5, 1.8, boost × normalizedRPM)

Layer 3: ブローオフバルブ (BOV)

スロットルを急閉した時に余剰ブースト圧を放出する音。2フェーズ合成。

// 発動条件
throttleDrop > 0.5 AND prevBoost > 0.4

// Phase 1 (0-0.15s): 初期エアバースト
env = exp(-t × 10)
sample = env × (hiss × 0.7 + BP_2kHz × 0.5)

// Phase 2 (0.15-0.6s): フラッター
flutterEnv = exp(-ft × 4)
pulse = |sin(ft × 25π)| > 0.6 ? 1 : 0
sample = env × pulse × (noise + tone_800Hz)

Phase 1は圧力放出の「プシュー」、Phase 2はバルブのフラッター振動。25Hzの矩形波パルスでバルブの断続的な開閉を模擬。

ブーストモデル

// 非対称追従 (spool up遅い, ブースト抜け速い)
targetBoost = throttle × Clamp01((rpm-0.2)/0.3)
rate = (target > current) ? 3 : 8
currentBoost = Lerp(current, target, dt × rate)

// ↑ ブースト上昇3Hz, 低下8Hz
// スプールアップ約0.3秒, 抜け約0.12秒

RPM 20%以下ではブーストゼロ（タービン未作動）。20-50%のゾーンでリニアに上昇し、50%以上でフルブースト。

💥 衝突サウンド COLLISION

CrashSound.cs - OnCollisionEnterで衝突相対速度を取得し、速度に比例した音量でワンショット再生。

速度-音量マッピング

relativeSpeed = collision.relativeVelocity.magnitude

if relativeSpeed ≥ 5 m/s:
  t = InverseLerp(5, 30, relativeSpeed)
  volume = Lerp(0.2, 1.0, t)
  PlayOneShot(crashClip, volume)

5m/s(18km/h)以下の軽い接触は無音。30m/s(108km/h)以上で最大音量。3Dサウンド(spatialBlend=1)で衝突位置から音が聞こえる。

設計意図

なぜPlayOneShotか: Playだと前の音が止まるが、PlayOneShotは重複再生可能。多重クラッシュ時に音が途切れない。

なぜInverseLerpか: 線形マッピングだと低速衝突の音量変化が不自然。InverseLerpで閾値内を0-1に正規化し、Lerpで最小音量0.2を保証。完全な無音衝突を避ける。

🏆 順位管理システム RANKING

RacePositionManager.cs - C#リフレクションで全AI車両の順位とブーストファクターをリアルタイム更新。レコーディングデータを共通基準軸として全車の進行度を統一的に計算。

レースゲーム開発者へ: 順位計算の最大の課題は「全車が同じコースを走っている保証がない」こと。このシステムでは、1台分のレコーディングデータ（約400-800ポイント）を基準軸として、全車の位置を最近傍探索で投影する。コースのショートカットや逆走も正しく検出できる。
参考: F1ゲームの順位計算でもセクターベースの進行度追跡が標準手法。

アーキテクチャ: Reflection Bridge

RecordingCPUControllerのprivateフィールドにReflectionでアクセス。キャッシュ済みFieldInfoで高速化。

// Start()でReflectionフィールドをキャッシュ
fPts = typeof(RecordingCPUController)
  .GetField("points", NonPublic | Instance)
fRank = ...GetField("currentRank", ...)
fBoost = ...GetField("rankBoostFactor", ...)

// Update()で読み書き (0.15秒間隔)
var pts = fPts.GetValue(car) as List<LapDataPoint>
fRank.SetValue(car, rank)
fBoost.SetValue(car, (rank-1)/(total-1))

進行度計算アルゴリズム

// 1. 最近傍点探索 (O(N) ブルートフォース)
idx = argmin(|pos - refPoints[i]|²)
// XZ平面距離のみ (Y無視)

// 2. ラップ検出 (75%→25%のラップアラウンド)
if prevIdx > N×0.75 AND curIdx < N×0.25:
  lapCount++

// 3. 進行度 (周回数 + コース内位置)
progress = lapCount + idx / refPointsCount

プレイヤー順位の決定

playerProg = UpdateProgress(playerId, pos)
aheadCount = count(cpuProg > playerProg)
playerRank = aheadCount + 1

// UI表示: "3rd / 17 Lap 2/8"

レース終了処理

if playerLap ≥ totalLaps:
  Time.timeScale = 0 // 一時停止
  ShowFinishResult(rank, total)
  // 3秒後に "title" シーンへ遷移

// 順位色: 1st=金, 2-3位=シアン, 他=白

💾 ラップレコーダー RECORDING

LapRecorder.cs - プレイヤーの走行データを20Hzでサンプリングし、AI走行の基盤データとしてJSON出力。

レースゲーム開発者へ: レコーディングAIの品質は記録データの品質に直結する。20Hz (0.05秒間隔) は、200km/h走行時でも約2.8m間隔のサンプリングとなり、Catmull-Rom補間と組み合わせれば十分な精度。60Hzにするとデータ量が3倍になるが、スプライン補間の精度向上は僅か。

velX, velY, velZ: float // 速度ベクトル
steerAngle: float // WheelCollider実値
motorTorque: float // 後輪トルク実値
